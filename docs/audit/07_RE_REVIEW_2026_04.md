## Methodology

This re-review followed the live-system safety model defined in epic `#194`: read-only by default, no production endpoint scanning, no live Stripe/Xero mutation traffic, and no dependency or schema changes except targeted remediation. The audit strategy was:

1. Re-verify the audited spine from 2026-04-08 through citation-level spot checks.
2. Deep-dive newly added post-launch surfaces: hashed-token migration validation, blue-green deploy automation, finance/Xero boundaries, kiosk PIN behavior, and post-launch public/email changes.
3. Use runtime drills only where explicitly safe: local or non-prod auth/path probes, production read-only SQL validation for the hashed-token migration, and webhook/path behavior confirmation without state mutation.
4. File each concrete finding as a separate GitHub issue, then close only after repo-backed remediation or explicit evidence was attached.
